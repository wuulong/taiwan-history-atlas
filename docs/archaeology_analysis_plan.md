# 🏛️ 臺灣考古遺址正規分析與厚化計畫 (Archaeological Analysis Plan)

## 📋 目標概述
針對目前已獲取的 2,563 筆臺灣考古遺址底冊（Master Registry），建立一套標準化的厚化程序。我們不只是要記錄「座標」，更要挖掘每個點位背後的「文化特徵」、「生存邏輯」與「時空地位」。

---

## 🏛️ 第一部分：分析維度架構 (Analysis Dimensions)

分析將分為四個正規學術層次，引導資料庫中 `meta_data` 的處理方向：

### 1. 物質特徵層 (Materiality & Artifacts)
*   **出土遺物分析**：陶器（繩紋、彩陶、黑陶）、石器（石鋤、石錛）、骨角器、金屬器、飾品。
*   **遺跡現況**：建築基址、墓葬分佈（葬式）、貝塚、灰坑（廢棄活動區）。
*   **分析關鍵**：從遺物組合判斷該聚落的「生產/生活型態」。

### 2. 時空脈絡層 (Chronology & Stratigraphy)
*   **年代精確化**：將 B.P. (Before Present) 資料映射至具體的文化期（如：大坌坑文化、圓山文化、蔦松文化）。
*   **文化繼承性**：識別多文化層 (Multicomponent) 遺址，分析同一空間在不同時代的利用情形。
*   **分析關鍵**：建立「時間軸地圖」，觀察文化的擴散與移轉。

### 3. 空間環境層 (Landscape & Settlement Pattern)
*   **微地形分析**：判斷遺址位處河階、海濱、沙丘、平原還是山區。
*   **資源鏈結**：分析遺址與水源（曾文溪等）、農地、石材來源的距離與可及性。
*   **空間關係**：分析遺址群的大小與疏密，判斷其是否為核心聚落、衛星部落或季節性營地。

### 4. 調查與研究層 (Academic Status)
*   **調查歷程**：紀錄自日治時期至當代普查的調查次數與深度。
*   **法律地位**：國定/市定/列冊/疑似之學術評點。
*   **文獻鏈結**：關聯至《臺灣考古遺址普查報告書》等正規出版品。

## 🚀 第二部分：資料演化階段藍圖 (The L0-L4 Evolution Path)

為了確保 2,500+ 點位能系統性地被厚化，我們將執行藍圖分為五個進化層級（Levels）：

### L0 (Raw Data) $\rightarrow$ L1 (Master Registry): 基礎架構佈建
*   **動作**：從各地 Open Data 與中研院系統爬取原始 JSON/CSV，並統一匯入 SQLite 資料庫 (`history_atlas.db`)。
*   **產出**：具備一致的 `site_id`, `name`, `latitude`, `longitude` 以及保留原始殘骸的 `meta_data` 欄位。
*   **狀態**：目前已完成 2,563 處遺址的 L1 收編。

### L1 $\rightarrow$ L2 (Feature Extraction): 屬性與標籤標準化

此階段的主要目標是將原始資料（L0）中雜亂的 `meta_data` JSON 轉化為結構化的知識標籤（L2），以便進行大數據分析與疊圖查詢。

在 Master Registry (`archaeological_sites` 資料表) 中，我們新增了以下四個關鍵欄位：

1.  **`cultural_period` (TEXT): 時空座標標準化**
    *   **邏輯**：分析 `year` 陣列中的 B.P. (距今) 年代數值。
    *   **映射意義**：
        *   `> 5000 B.P.` $\rightarrow$ 舊石器/新石器早期
        *   `5000 - 3500 B.P.` $\rightarrow$ 新石器中期 (如大坌坑文化)
        *   `3500 - 2000 B.P.` $\rightarrow$ 新石器晚期 (如圓山、卑南文化)
        *   `< 2000 B.P.` $\rightarrow$ 金屬器時代 (如十三行、蔦松文化)

2.  **`is_multicomponent` (BOOLEAN): 多文化層識別**
    *   **邏輯**：判斷該遺址的年代是否跨越兩個以上的時期（例如同時包含新石器晚期與金屬器時代）。
    *   **映射意義**：若為 `True`，代表古人在不同歷史階段皆選擇在此地定居，顯示該地點具備極高的長期生存或戰略價值。

3.  **`importance_rank` (INTEGER): 學術與法規權威評分**
    *   **邏輯**：根據政府或中研院的評級敘述給予測重。
    *   **映射意義**：
        *   **Rank 1**：指定遺址 (國定/市定)，具最高保存價值與最完整調查。
        *   **Rank 2**：重要遺址 / 建議指定。
        *   **Rank 3**：一般性遺址 / 列冊遺址，數量最多。
        *   **Rank 4**：疑似遺址 / 孤立地點，考古證據尚待確認。

4.  **`site_function` (JSON): 生存語法與地理特徵提取**
    *   **邏輯**：利用關鍵字比對 (Regex) 在描述與原始 JSON 中進行探勘。
    *   **映射意義**（含功能與微地形）：
        *   `貝塚`、`墓葬`、`聚落遺構` (如：柱洞)、`物質遺留` (如：陶片、石器)。
        *   `台地/高處`、`濱溪`、`沿海/沙丘`。
        *   可快速藉由此欄位篩選出「濱海的貝塚遺址」或「山區的墓葬區」。

*   **動作**：利用 `feature_extractor.py` 清洗這 2,563 處點位，並完成標籤註記。
*   **狀態**：已全面完成 2,563 處遺址的 L2 標籤洗鍊。

### L2 $\rightarrow$ L3 (Semantic Enrichment): 學術語意厚化
*   **動作**：利用大語言模型 (LLM / RAG) 結合內建考古學規則，針對「文化時期」與「特徵標籤」自動產生約 200 字的歷史脈絡散文。
*   **產出**：將高品質的「大歷史與地理脈絡摘要」寫入 `historical_context` 鍵值，讓非專業讀者也能理解遺址的歷史重要性。
*   **狀態**：已於曾文溪流域進行 3 處重點遺址的 POC，證實成效極佳。即將展開批次厚化。

### L3 $\rightarrow$ L4 (Relational Graph & Spatial Intelligence): 關聯網路與空間拓樸
*   這是知識圖譜的最頂層，將孤立的點連成「網 (Network)」與「面 (Surface)」。
*   **動作與產出**：
    1.  **河流分群與最近鄰空間運算 (Tributary Clustering & Spatial Topology)**：
        *   不再只看單一主流，而是將點位與整個水系網路 (流域/所有主支流) 進行疊加。
        *   **最近水系綁定 (Nearest Neighbor)**：透過 GIS 演算法自動計算遺址與周遭所有次流域（如：菜寮溪、官田溪）的最短直線距離，判定其真正的「母親河」。
        *   **交會點識別 (Confluence Hub Detection)**：若一遺址同時距離多條河川低於特定閾值（如 < 2km），自動標記為水運或資源的「樞紐聚落Hub」。
    2.  **同源網路 (Cultural Diffusion)**：以空間距離為半徑（如 20km），自動將同屬「蔦松文化」的遺址進行連線 (Edges)，找出區域核心點與衛星營地。
    3.  **跨域實體鏈結 (Entity Linking)**：將 `meta_data` 中出土的文物代碼（如中研院標本 ID）自動關聯至外部影像圖庫；或將遺址座標與相同時空的「歷史事件（如西拉雅族事件）」進行時空疊加。
*   **目標**：建立「聚落動態演變圖」，高度還原古人群的勢力範圍、跨水系真實生活圈，以及對河川頻繁改道的適應史。

## 📈 評估指標
*   **完備率 (L2)**：2,000+ 點位皆具備標準化文化期標籤（已達成）。
*   **厚化度 (L3)**：重點區域（如曾文溪流域）的 100+ 重點遺址具備高品質 L3 歷史摘要。
*   **網絡度 (L4)**：成功建立曾文溪水脈與各大遺址群的 GIS 拓樸關聯圖譜。
